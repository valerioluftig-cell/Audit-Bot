Architecture
Five-stage pipeline

Each parish PDF passes through five sequential stages. Extraction and detection run locally with no API calls. Claude is invoked once per parish with the assembled statement text, and results are cached to disk so re-runs skip the API entirely.

Stage 2
Page detection

The core challenge is unreliable OCR from scanned documents. The detector uses layered logic: OCR-variant keyword groups that must all match, exclude lists that reject lookalike pages, a notes-page guard that inspects only the title area, and a confirmation window that verifies data content before committing to an anchor page.

Notes-page guard — title area only, not footnote footers pipeline/extractor.py 
def _is_notes_page(text: str) -> bool:
    # Notes title is always in the first ~200 chars (parish name + title + date).
    # Checking only the top means a "see notes to financial statements" footer
    # on a CBS page does NOT fire this guard and stop collection early.
    first400 = text[:400]
    very_top = text[:200]
    return (
        "notes to financial"   in first400
        or "notes to the financial" in first400
        or "notes to the basic"    in first400
        or ("notes to the" in very_top and "statements" in very_top)
        or ("notes to"     in text[:150] and "statements" in text[:150])
    )

Core matcher — TOC guard + exclude check + require_any logic pipeline/extractor.py 
def _matches_any(text: str, patterns: dict, allow_notes: bool = False) -> bool:
    if not allow_notes and _is_notes_page(text):
        return False   # notes title in first 200 chars → not a primary statement

    # Table of contents lists every statement name but contains no data.
    # Dotted leaders (10+ periods) uniquely identify TOC formatting.
    if "table of contents" in text[:400] or "........." in text:
        return False

    for excl in patterns.get("exclude", []):
        if excl in text:
            return False   # MDA, reconciliation, combining, fiduciary, etc.

    # AND within a group, OR across groups — any one full match is enough
    for req_group in patterns.get("require_any", []):
        if all(kw in text for kw in req_group):
            return True
    return False

Engineering challenge
OCR corruption

Louisiana parish audits are scanned documents. OCR produces systematic corruptions — letter substitutions, word merges, inserted punctuation — that vary by scanning hardware. For most parishes, adding a new pattern variant resolves the issue. For others (Orleans), the corruption is too severe for any pattern and requires a manual page override.

Why MDA exclusion matters: Management Discussion & Analysis pages describe capital assets and revenues in narrative form. Their text overlaps heavily with actual statements — the same keywords appear but in sentences rather than table headers. Without MDA_EXCLUDE and the notes-page guard, these pages would anchor the detector and the wrong content would be sent to Claude.

Output
The four financial statements

Every Louisiana parish audit contains these four statements. The pipeline produces a separate Excel workbook for each, with one worksheet per parish for cross-sectional comparison.

CBS
Combined Balance Sheet
Governmental funds view. Each major fund (General, Debt Service, Capital Projects, etc.) is a column. Shows cash and receivables held, payables and deferred inflows owed, and fund balance broken down by spendability: nonspendable, restricted, committed, assigned, unassigned.
Cash & deposits Taxes receivable Accounts payable Deferred inflows Fund balances by type
SOA
Statement of Activities
Government-wide income statement. Each function shows total expenses, program revenues earned back (charges, grants), and the net cost covered by general revenues — property taxes, sales taxes, investment income. Ends with change in net position.
Expenses by function Charges for services Operating & capital grants Property taxes Sales taxes Net position change
SONA
Statement of Net Position
Government-wide balance sheet. Unlike the CBS, includes capital assets (infrastructure, buildings, equipment) and long-term liabilities (bonds, pension obligations). Net position = assets + deferred outflows − liabilities − deferred inflows.
Capital assets net Long-term liabilities Deferred outflows Deferred inflows Net position components
CA
Capital Assets Schedule
Tracks each asset category from beginning to ending balance: land, construction in progress, buildings, machinery, infrastructure. Shows additions, disposals, and annual depreciation. May appear as a primary statement or embedded in the Notes section.
Land Infrastructure Buildings Additions / Disposals Accumulated depreciation
Stage 1 & 5
Web interface

FastAPI accepts batch PDF uploads, runs the pipeline in a background thread, and streams each log line to the browser via Server-Sent Events. Download buttons appear when all four workbooks are ready.

Background job + SSE stream — each log line pushed to browser in real time webapp/app.py 
@app.post("/run")
async def run(files: list[UploadFile] = File(...), year: int = Form(...)):
    job_id = str(uuid.uuid4())[:8]
    q: queue.Queue[str] = queue.Queue()
    JOBS[job_id] = {"queue": q, "status": "running"}

    def _run():
        run_pipeline(..., progress_callback=lambda msg: q.put(msg))
        q.put("__DONE__")   # sentinel closes the SSE stream

    threading.Thread(target=_run, daemon=True).start()
    return {"job_id": job_id}


@app.get("/progress/{job_id}")
def progress(job_id: str):
    def stream():
        q = JOBS[job_id]["queue"]
        while True:
            try:
                msg = q.get(timeout=25)
            except queue.Empty:
                yield "data: \n\n"   # heartbeat keeps connection alive
                continue
            yield f"data: {json.dumps(msg)}\n\n"
            if msg == "__DONE__": break

    return StreamingResponse(
        stream(), media_type="text/event-stream",
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"},
    )
